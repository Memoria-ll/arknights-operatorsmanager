# Arknights Operators Manager 要件定義書（改訂版）

## 1. システム概要
アークナイツのオペレーター育成を効率的に管理し、育成計画と素材過不足を一元管理するアプリケーション。

---

## 2. 開発環境・技術要件

### 2.1 開発環境
- 対象OS: Windows 11以降のデスクトップアプリケーション（NFR-CP-001/002、CONST-011参照）。
- 言語／ランタイム: C# / .NET 8。
- UIフレームワーク: WPF（MVVM）。MVVMライブラリは CommunityToolkit.MVVM を使用する。
- 内蔵ブラウザ: WebView2（Microsoft Edge WebView2）。ペンギン急便・Wiki等の表示に用いる（FR-OP-026、FR-SY-030）。
- 一覧表示: 大量行（400体以上）はDataGrid等のUI仮想化で表示する（NFR-PF-001）。
- テスト: xUnit。view/logic seam（ViewModel・サービス等のロジック単位）を単体テスト対象とする。
- 配布／自己更新: インストーラ（MSI/MSIX）は使用しない。Velopack を用い、GitHub Releases から管理者権限不要・差分更新で自己更新する（3.4.3）。完全オフラインで動作すること（3.6.2、CONST-005）。

### 2.2 技術制約
- オフライン動作を基本とする
- インターネット接続は更新チェック時のみ必要
- データ永続化はJSON形式で行う

---

## 3. 機能要件

### 3.1 オペレーター管理機能

#### 3.1.1 状態管理
**概要**: オペレーターの現在状態と目標状態を管理する

**入力項目**
- 昇進段階（0/1/2）
- レベル（レアリティと昇進段階により上限が異なる）
- スキルレベル（1～7）
- スキル特化レベル
  - スキル1特化（0～3）
  - スキル2特化（0～3）
  - スキル3特化（0～3）
- モジュールレベル
  - モジュールX（0～3）
  - モジュールY（0～3）
  - モジュールD（0～3）
  - モジュールA（0～3）
- 潜在（0～6、0=未所持、1以上=所持）
- 信頼度（フリーテキスト入力）
- Paradoxクリア（クリア済み/未クリア）
- グループ(オペレータに紐づく現在/目標共通情報)

**状態の種類**
- **現在状態**: オペレーターの現在の育成状況
- **目標状態**: ユーザーが目指す育成目標

**機能要件**
- FR-OP-001: 各オペレーターの現在状態を入力・保存できること
- FR-OP-002: 各オペレーターの目標状態を入力・保存できること
- FR-OP-003: 現在状態と目標状態を個別に管理できること
- FR-OP-004: 入力された状態データを永続化できること
- FR-OP-005: オペレーター単位でグループ・メモを設定・編集できること（現在/目標で共通の情報として扱う）
- FR-OP-006: グループの作成・リネーム・削除を行い、オペレーターをいずれか1つのグループに割り当てできること（単一所属）
- FR-OP-007: グループ・メモを永続化すること

#### 3.1.2 表示機能

##### 3.1.2.1 テーブル表示モード
**概要**: オペレーター一覧をテーブル形式で表示し、カスタマイズ可能な列構成を提供する

**テーブル表示対象項目（この範囲からユーザーが選択）**
- 基本情報: オペレーター名、レアリティ、職業
- 育成情報: 昇進段階、レベル、スキルレベル、スキル特化レベル、モジュールレベル、信頼度
- グループ情報: グループ、メモ

**機能要件**
- FR-OP-010: オペレーター一覧をテーブル形式で表示できること
- FR-OP-011: テーブル表示の対象は「基本情報」「育成情報」「グループ情報」とすること
- FR-OP-012: ユーザーは上記の表示対象項目の中から、表示したい列を自由に選択できること
- FR-OP-013: 列の表示順序を変更できること（ドラッグ＆ドロップまたは設定画面）
- FR-OP-014: 職業別・レアリティ別にグループ化表示できること
- FR-OP-015: 各列でソート（昇順/降順）ができること
- FR-OP-016: 列の表示設定をユーザー設定として保存できること

##### 3.1.2.2 詳細表示モード（開閉式）
**概要**: テーブル行をクリックすることで、そのオペレーターの全情報を展開表示する

**表示情報（すべて表示、ユーザー選択不可）**
- 基本情報: 名前、レアリティ、職業、職分、種族、所属、性別、出身
- 実装情報: 実装日、パラドックス実装日、モジュール実装日
- 育成情報: 昇進段階、レベル、スキルレベル、スキル特化レベル、モジュールレベル
- その他: ポテンシャル、信頼度、グループ、メモ

**機能要件**
- FR-OP-020: テーブル行をクリックすることで、そのオペレーターの詳細情報を展開表示できること
- FR-OP-021: 展開された詳細情報には、すべてのオペレーター情報を表示すること（ユーザーによる表示項目選択なし）
- FR-OP-022: 再度クリックすることで詳細情報を折りたたむことができること
- FR-OP-023: 複数のオペレーターの詳細を同時に展開できること
- FR-OP-024: 詳細表示内で現在状態・目標状態の両方を確認できること
- FR-OP-025: 詳細表示内でテーブル表示の情報も含めて表示すること
- FR-OP-026: 詳細表示から、オペレーターの `page` 番号に基づき内蔵ブラウザで該当Wikiページ（`https://wiki3.jp/arknightsjp/page/<page>`）を開けること

#### 3.1.3 入力検証機能
**概要**: ゲームルールに基づいた入力制約をチェックし、誤入力を防止する

**検証ルール**
- レアリティによる昇進段階制限
  - ★3: 昇進1まで
  - ★4以上: 昇進2まで
- 昇進段階によるレベル上限（レアリティ × 昇進段階。0は昇進不可）

  | レアリティ | 未昇進(昇進0) | 昇進1 | 昇進2 |
  |---|---|---|---|
  | ★1 | 30 | 0 | 0 |
  | ★2 | 30 | 0 | 0 |
  | ★3 | 40 | 55 | 0 |
  | ★4 | 45 | 60 | 70 |
  | ★5 | 50 | 70 | 80 |
  | ★6 | 50 | 80 | 90 |
- 昇進段階による機能制限
  - スキル特化: 昇進2以降のみ
  - モジュール: 昇進2以降のみ
- スキル・モジュールの存在チェック
  - オペレーターごとに保有するスキル数が異なる
  - オペレーターごとに実装されているモジュールが異なる

**機能要件**
- FR-OP-030: レアリティに応じた昇進段階の上限チェックを行うこと
- FR-OP-031: 昇進段階とレアリティに応じたレベル上限チェックを行うこと
- FR-OP-032: 昇進段階に応じた機能利用可否チェックを行うこと（スキル特化、モジュール）
- FR-OP-033: オペレーターが保有しないスキル・モジュールへの入力を防止すること
- FR-OP-034: 実装日フィルタリングにより、未実装要素への入力を防止すること
- FR-OP-035: 検証エラー時には、わかりやすいエラーメッセージを表示すること
- FR-OP-036: リアルタイム検証により、入力時に即座にエラーを通知すること
- FR-OP-037: 各育成項目で「現在値 ≤ 目標値」を維持すること。現在値が変化した際（強化実行入力を含む）は自動でこのガードを実行し、目標値を下回らないよう整合させること

#### 3.1.4 別形態オペレーター（コードに「_」を含む）
**概要**: 同一オペレーターの別形態（例: 近衛アーミヤ）を管理する

**コード規則**
- 別形態のコードは「親コード_サフィックス」の形式（例: `R001_2`）
- 親コードは「_」より前の部分（例: `R001`）

**機能要件**
- FR-OP-040: 別形態オペレーターは独立したエントリとして管理すること
- FR-OP-041: 別形態オペレーターの表示コードは「_」以降を除いた親コードで表示すること
- FR-OP-042: 別形態オペレーターの昇進欄・レベル欄・信頼度欄は無効化（編集不可）すること
- FR-OP-043: 別形態オペレーターのスキル・特化のバリデーション時は、親コードのオペレーターの昇進・レベルを参照すること
- FR-OP-044: 親オペレーターが未所持の場合、別形態オペレーターの育成入力を制限すること

#### 3.1.5 異格オペレーター（alternates）
**概要**: 同一キャラクターの別オペレーター間で信頼度を共有する

**データ構造**
- `alternates`フィールドに、同一キャラクターの全オペレーターコードをリストで保持
- 自分自身のコードも含む（例: `["LM04", "R112", "YD30"]`）

**機能要件**
- FR-OP-050: 異格オペレーター間で信頼度を双方向同期すること
- FR-OP-051: いずれかのオペレーターの信頼度を変更した場合、他の異格オペレーターにも同じ値を反映すること
- FR-OP-052: 昇進・レベル・スキル等の育成状態は各オペレーター独立で管理すること
- FR-OP-053: 異格オペレーター間の関連を視覚的に表示すること（任意）
- FR-OP-054: 異格オペレーター間の信頼度共有機能をON/OFF切り替えできること
- FR-OP-055: 信頼度共有がOFFの場合、各オペレーターの信頼度を個別に編集し、相互に同期しないこと
- FR-OP-056: 信頼度共有のON/OFF状態を設定として永続化すること

#### 3.1.6 無料育成オペレーター（freeGrowth）
**概要**: 育成に素材を一切必要としないオペレーター（統合戦略オペレーター等）を管理する

**特徴**
- 昇進、レベル、スキルレベル等の育成要素は他のオペレーターと同様に存在する
- ただし育成に必要な素材（龍門幣・経験値含む）が全て不要
- ゲーム内コンテンツのプレイで自動的に成長する

**機能要件**
- FR-OP-060: 無料育成オペレーターは `freeGrowth: true` フラグで識別すること
- FR-OP-061: 無料育成オペレーターの必要素材計算では、昇進・スキル・特化・モジュールコストを0とすること
- FR-OP-062: 無料育成オペレーターのレベルアップコスト（龍門幣・経験値）を0とすること
- FR-OP-063: 無料育成オペレーターは素材計算の対象外とし、集計から除外すること

#### 3.1.7 オペレーター検索機能
**概要**: オペレーター名でオペレーターを検索し、該当オペレーターへ素早くアクセスする

**機能要件**
- FR-OP-070: オペレーター名でオペレーターを検索できること
- FR-OP-071: キーボードショートカット `Ctrl+F` で検索を起動できること

#### 3.1.8 目標一括入力
**概要**: 複数オペレーターの目標値を一括で入力する

**入力仕様**
- 対象レアリティ範囲を選択する（「いくつから」「いくつまで」）
- 各育成項目（昇進段階、レベル、スキルレベル、スキル特化、モジュール、信頼度 等）ごとに「反映する/しない」を切り替えできる
- 「反映する」にチェックした項目の値を、対象オペレーターの目標状態に一括入力する
- オプションスイッチ「現在値にも同じ値を入力」: OFF=目標のみ反映、ON=現在値にも同一の値を反映する

**機能要件**
- FR-OP-080: 対象レアリティ範囲（from～to）を選択できること
- FR-OP-081: 各育成項目ごとに反映する/しないを切り替えできること
- FR-OP-082: 反映ONの項目の値を、対象オペレーターの目標状態に一括入力できること
- FR-OP-083: オプションスイッチONの場合、同一の値を現在状態にも一括入力できること（OFFの場合は現在状態に反映しない）
- FR-OP-084: 一括入力後も「現在値 ≤ 目標値」ガード（FR-OP-037）を満たすこと

#### 3.1.9 フィルタ機能
**概要**: 各種条件でオペレーターの表示を絞り込む

**フィルタ条件**
- 基本情報: 職業、職分、性別、出身地 等
- 実装日（大陸版 / グローバル版を選択）: 日付を選択し、since（以降）/ to（以前）で範囲指定する
  - 対象日付はオペレーター実装日に加え、モジュール実装日・逆理演算（Paradox）実装日を含む
- 状態: 所持/未所持、目標達成/未達成

**機能要件**
- FR-FL-001: 職業・職分・性別・出身地 等の基本情報でフィルタできること
- FR-FL-002: 実装日（大陸版/グローバル版を選択）について、日付の since/to による範囲フィルタができること
- FR-FL-003: 実装日フィルタの対象に、オペレーター実装日・モジュール実装日（X/Y/D/A）・逆理演算実装日を含め、日付を持つ各要素を個別に判定すること
- FR-FL-004: 所持/未所持でフィルタできること
- FR-FL-005: 目標達成/未達成でフィルタできること
- FR-FL-006: 複数の条件を組み合わせてフィルタできること
- FR-FL-007: フィルタ条件をプリセットとして保存し、呼び出して再適用できること
- FR-FL-008: 保存したフィルタプリセットを編集・削除できること

#### 3.1.10 育成進捗ダッシュボード（低優先）
**概要**: 現在状態と目標状態の差から、育成の達成度を集計して可視化する

> ※ 「達成」の定義（全項目一致＝達成 等）は未定（hold）。設計時に決定する。

**機能要件**
- FR-OP-090: 全体の目標達成率（現在状態が目標状態に到達している割合）を集計・表示できること
- FR-OP-091: レアリティ別・職業別等の区分で達成率を表示できること
- FR-OP-092: 進捗の集計はフィルタ・グループの絞り込みと連動できること

#### 3.2.1 所持素材管理
**概要**: ユーザーが保有する素材の数量を管理する

**機能要件**
- FR-MT-001: 各素材の所持数を手動で入力・更新できること
- FR-MT-002: 素材一覧を表示できること
- FR-MT-003: 素材の所持数をリアルタイムで更新・表示できること
- FR-MT-004: 所持素材データを永続化できること

#### 3.2.2 必要素材自動計算
**概要**: オペレーターの現在状態から目標状態への育成に必要な素材を自動計算する

**計算対象**
- 昇進に必要な素材・龍門幣
- レベルアップに必要な経験値と龍門幣
- スキルレベルアップに必要な素材
- スキル特化に必要な素材
- モジュール強化に必要な素材

**コスト算出データの所在**
- レベルアップの経験値・龍門幣コスト（昇進段階別・レベル別）は、コード内に定義する固定テーブルから算出する。マスターJSONには含めない。
  - 参照データ: `expdata.csv`（STEP=昇進段階0/1/2、ID=2000:龍門幣 / 1000:経験値、列=レベル1～90の累計値）。
- 昇進・スキル特化・モジュール強化に必要な龍門幣は、マスターデータ（operator_master_data）側のコスト定義に追記する（後述 FR-VR-016）。

**機能要件**
- FR-MT-010: オペレーターの現在状態→目標状態の差分から必要素材を自動計算できること
- FR-MT-015: レベルアップコスト（経験値・龍門幣）はコード内定義の固定テーブル（`expdata.csv`相当）から算出すること
- FR-MT-016: 昇進・特化・モジュールに必要な龍門幣をマスターデータのコスト定義から算出し、必要素材計算に含めること
- FR-MT-011: 計算結果を素材種別ごとに集計できること
- FR-MT-012: 複数オペレーターの必要素材を合算して表示できること
- FR-MT-013: 計算結果をカテゴリ別（昇進用、レベル用、スキル用等）に分類表示できること
- FR-MT-014: 各素材について「どのオペレーターが」「何のために（昇進/レベル/スキル等）」必要としているかの詳細情報を確認できること

#### 3.2.3 不足素材可視化
**概要**: 所持数と必要数を比較し、不足している素材を視覚的に強調表示する

**機能要件**
- FR-MT-020: 素材ごとに「所持数」「必要数」「過不足」を表示できること
- FR-MT-021: 不足している素材を赤字やハイライトで強調表示できること
- FR-MT-022: 「あと○○個足りない」という形式で不足数を明示できること
- FR-MT-023: 十分に所持している素材と不足素材を区別して表示できること

#### 3.2.4 加工割合機能
**概要**: 素材の加工（合成）使用割合を設定し、実際に育成に使用する素材数を調整する。加工する分は、その素材を直接消費する代わりに、マスターデータの合成レシピ（material の `recipe`）に従って下位素材を消費する。下位素材がさらに `recipe` を持つ場合は、最下位素材まで再帰的に展開して消費する。

**機能要件**
- FR-MT-030: 各素材に対して加工割合（0～100%）を設定できること
- FR-MT-031: 加工割合を考慮した必要数（加工後必要数 = 必要数 × 加工割合 / 100）を計算できること
- FR-MT-032: グローバル加工割合を設定し、全素材に一括適用できること
- FR-MT-033: 不足数・余り数は加工割合を考慮して計算すること
- FR-MT-034: 加工する分（必要数 × 加工割合 / 100）は、当該素材を直接消費する代わりに `recipe` で定義された下位素材を消費するものとして、下位素材の必要数に加算すること。下位素材がさらに `recipe` を持つ場合は最下位素材まで再帰的に展開すること（`recipe` を持つ素材のみ対象）

#### 3.2.5 素材加工表
**概要**: 加工元（合成レシピ）を持つ上位素材について、その加工に必要な下位素材数を一覧表形式で表示する。メイン画面とは別ウィンドウで提供する。

**機能要件**
- FR-MT-040: 合成レシピ（`recipe`）を持つ上位素材と、その加工に必要な下位素材・必要数をテーブル形式で表示できること
- FR-MT-041: 素材加工表をメイン画面とは別ウィンドウで表示できること

#### 3.2.6 素材入手登録
**概要**: 入手した素材を登録し、所持数（BaseOwned）を更新する。入手区分として「入手」「加工」を選択できる。登録対象は編集可能な入手登録リストに集約し、確定（登録）時にまとめて所持数へ反映する。

**入手区分**
- **入手**: 選択した素材を無条件で所持数に加算する
- **加工**: 合成レシピに従い、選択した上位素材を所持数に加算し、必要な下位素材を所持数から減算する

**入手登録リストへの追加手段**
- 個別追加: 素材を選択して追加する
- ステージ一括追加: ステージを選択し、当該ステージで入手可能な素材をまとめて追加する

**ステージ別入手可能素材データ**
- ステージで入手可能な素材は、ペンギン急便データ（GitHub: `ArknightsToolOperatorsManager/penguin-stats-data`）を定期的に取得して解決する。取得データはローカルに保持し、オフラインでも参照できること。
- 一括追加の対象は、当該ステージのドロップ素材のうち、本アプリの素材ID（material_master）と一致する素材のみとする（一致しないIDは対象外）。
- ステージの識別・表示は、ステージマスター（`stageId`・略号・多言語ステージ名）に基づく（JSON化予定）。
- 多言語ステージ名が未確定（大陸版先行でグローバル版未実装）のステージは、中国語名で代替表示する。

**機能要件**
- FR-MT-050: 素材を選択し、入手区分（入手/加工）と個数を指定して入手登録リストに追加できること
- FR-MT-051: 入手の場合、選択素材の所持数を入力個数分加算すること
- FR-MT-052: 加工の場合、上位素材の所持数を入力個数分加算し、`recipe` に従って下位素材の所持数を減算すること
- FR-MT-053: 入手登録リストの各行で入手区分・個数を編集でき、行を削除でき、確定（登録）時にリスト内容をまとめて所持数へ反映できること
- FR-MT-054: ステージを選択し、当該ステージのドロップ素材のうち本アプリの素材ID（material_master）と一致する素材を入手登録リストへ一括追加できること
- FR-MT-055: ステージ別の入手可能素材データはペンギン急便データ（GitHub: `ArknightsToolOperatorsManager/penguin-stats-data`）を他のマスターデータと同じタイミング（FR-VR-019）で取得して用い、取得データはローカルに保持してオフラインでも参照できること
- FR-MT-056: ステージはステージマスター（`stageId`・略号・多言語名）に基づき、多言語のステージ名で選択できること
- FR-MT-057: 多言語ステージ名が未確定（大陸版先行）のステージは、中国語名で代替表示すること

> ※ 入手登録は 3.2.1 所持素材管理の所持数（BaseOwned）を更新する。永続化は FR-MT-004 に従う。

#### 3.2.7 計算対象の絞り込み

- FR-MP-016: 必要素材自動計算のインプットとして、利用可能素材数を使用すること

#### 3.3.4 素材計画機能のUI制御

**機能要件**

- FR-MP-032: 素材計画機能が無効の場合、素材パネルの所持数欄でBaseOwnedを直接編集できること
- FR-MP-033: 素材計画機能が有効の場合、素材パネルの所持数欄は読み取り専用とし、利用可能素材数（BaseOwned + 計画入手分の合計）を表示すること。BaseOwnedの編集はInventoryウィンドウで行うこと

**概要**: 新オペレーター実装時などに、マスターデータを自動更新する

**取得元**

**更新対象**
- オペレーター基本情報（レアリティ、職業、職分、種族、所属、性別、出身、page等）

**機能要件**

- FR-VR-012: 新オペレーター・新素材・新モジュール等の差分更新ができること

- FR-VR-015: 更新履歴をログとして記録すること

#### 3.6.4 外部サイト連携（ペンギン急便 / Penguin Statistics）
**概要**: ペンギン急便（https://penguin-stats.io/）と連携し、所持素材データを当該サイトのインポート形式でエクスポート（コピー）できるようにする

**インポート形式（ペンギン急便 Planner config）**
- ルート: `{"@type":"@penguin-statistics/planner/config","items":[...],"options":{...},"excludes":[]}`
- `items` 各要素: `{"id":"<素材ID>","have":<所持数>,"need":<必要数>}`
  - `id`: 素材ID（文字列）
  - `have`: 所持数（利用可能素材数）
  - `need`: 必要数（不足分）
- `options`: `{"byProduct":false,"requireExp":false,"requireLmb":false}`
- `excludes`: `[]`

```json
{"@type":"@penguin-statistics/planner/config","items":[{"id":"30125","have":100,"need":12},{"id":"30115","have":94,"need":8}],"options":{"byProduct":false,"requireExp":false,"requireLmb":false},"excludes":[]}
```

**機能要件**
- FR-SY-030: ペンギン急便のページ（https://penguin-stats.io/）を内蔵ブラウザで開けること
- FR-SY-031: 所持素材データを上記のペンギン急便インポート形式（JSON）で生成し、コピーボタンでクリップボードにコピーできること（テキスト表示ではなくコピーボタンで実装すること）
- FR-SY-032: コピー対象は素材ID 10000以上の素材のみとすること
- FR-SY-033: 各素材の `have` に所持数（利用可能素材数）、`need` に必要数を設定すること
- FR-SY-034: `options` は `byProduct/requireExp/requireLmb` をいずれも `false`、`excludes` は空配列で出力すること

#### 3.6.5 通知ウィンドウ
**概要**: 1行程度のテキスト表示欄を横に並べた2つの通知ウィンドウを設ける。各ウィンドウは2つの機能を持ち、優先度に応じて表示を切り替える。可能な限りポップアップを使わず、これらの通知ウィンドウで情報を提示する。

**レイアウト**
- 通知ウィンドウ1（幅: 広い）: エラー通知 / tips
- 通知ウィンドウ2: セーブ通知 / スキル特化（訓練室）進捗
- 2つの欄を横並びに配置し、ウィンドウ1をウィンドウ2より広くする

**通知ウィンドウ1（エラー通知 / tips）**
- 表示優先度: エラー通知（上位） > tips（下位）
- エラーが発生していない間は、アプリ機能を1文程度で紹介するtipsを巡回表示する
- 重大なエラー発生時はエラー内容を表示し、tipsより優先する

**通知ウィンドウ2（セーブ通知 / スキル特化進捗）**
- 表示優先度: セーブ通知（一時表示） > スキル特化進捗
- セーブ完了時はセーブ完了通知を数秒表示して消える
- セーブ通知が無いときは、スキル特化の進捗をプログレスバー風に表示する。進行中の特化が無ければ「訓練室は空です」と表示する

**機能要件**
- FR-NT-001: 1行程度のテキスト欄を持つ通知ウィンドウを2つ設け、横に並べて配置すること（ウィンドウ1を広くする）
- FR-NT-002: 可能な限りポップアップを使用せず、通知ウィンドウで通知を行うこと
- FR-NT-010: 通知ウィンドウ1にエラー通知とtipsを表示すること
- FR-NT-011: エラーが発生していない間は、アプリ機能を1文程度で紹介するtipsを巡回表示すること
- FR-NT-012: 重大なエラー発生時はエラー内容を通知ウィンドウ1に表示し、tipsより優先（上位）すること
- FR-NT-020: 通知ウィンドウ2にセーブ通知とスキル特化進捗を表示すること
- FR-NT-021: 自動セーブ・手動セーブの完了時にセーブ完了通知を表示し、数秒後に自動的に消すこと
- FR-NT-022: セーブ通知が表示されていないときは、スキル特化の進捗をプログレスバー風に表示すること
- FR-NT-023: 進行中のスキル特化が無いときは「訓練室は空です」と表示すること
- FR-NT-024: スキル特化の完了時には通知を行わないこと

#### 3.6.6 ツールバー・設定
**概要**: アプリ操作の起点となるツールバーを設け、各機能へアクセスする。一部の設定はツールバーから開く設定画面に集約する。

**設定画面に集約する項目**

- 異格オペレーターの信頼度共有 ON/OFF（3.1.5）

※ 加工割合は素材欄で設定する（3.2.4）、素材計画機能のON/OFFは素材パネル/Inventoryウィンドウで操作する（3.3.4）。これらは設定画面には集約しない。

**機能要件**
- FR-SY-040: アプリ操作の起点となるツールバーを設けること
- FR-SY-041: ツールバーから開く設定画面に、表示言語・信頼度共有ON/OFF・大陸版情報表示ON/OFF を集約すること
- FR-SY-042: 設定画面で変更した設定を永続化すること
- FR-SY-043: 加工割合・素材計画機能のON/OFFは設定画面に集約せず、それぞれ素材欄・素材パネル/Inventoryウィンドウで操作すること

**機能要件**

**機能要件**

#### 3.6.9 操作の取り消し（Undo）
**概要**: 直前の操作を取り消す。アプリ全体で利用できることを目標とする。

**機能要件**
- FR-SY-070: 直前の操作を取り消し（Undo）できること（アプリ全体を対象とすることを目標とする）
- FR-SY-071: 少なくとも目標一括入力・強化実行（素材消費を伴う操作）は取り消し対象とすること
- FR-SY-072: 取り消し可能な操作には、操作前の状態（対象の数値・所持素材数等）への復元を含めること

> ※ 全操作のUndoが実装上困難な場合は、対象を一括入力・強化実行（FR-SY-071）に限定してよい。

### 3.9 育成実行・シミュレーション機能

#### 3.9.1 スキル特化機能
**概要**: 特定オペレーターに対して、プレイヤーが実際にゲーム内で実行した強化（昇進、レベル、スキルレベル、スキル特化、モジュール）を入力し、確定時に必要素材を所持数から即時消費して対象オペレーターの現在状態を即時反映する。スキル特化については、訓練時間を計算し通知ウィンドウ2（3.6.5）にプログレスバー風で進捗表示する。

**入力仕様（共通）**
- 対象オペレーターを選択する
- 各強化項目の「左の数値（現在値）」は入力対象ではなく現在値の表示であり、ユーザーは「右の数値（強化後にどこまで上げたか）」を入力する
- 入力対象: 昇進段階、レベル、スキルレベル、スキル特化（1/2/3）、モジュール

**スキル特化の追加入力**
- スキル番号（1/2/3）と特化レベル（0→1 / 1→2 / 2→3）を選択する
- 訓練時間（時間短縮率。例: 50% 等の%）を設定する。数字のフリー入力とし、可能ならスピンボタン（updown）を備える
- 減少時間（例: -50% 等の%）を設定する。同様に数字フリー入力＋可能ならスピンボタン

**訓練時間の算出**
- 訓練時間はテーブル定義せず、計算式で算出する。
- 基準時間（短縮なし）: 特化1=8:00:00、特化2=16:00:00、特化3=24:00:00（0→3 通算は 48:00:00）。
- 設定した訓練時間短縮率・減少時間率を基準時間に適用して算出する。

> 参考（実装は計算式。テーブルは定義しない）

  | 特化段階 | 0% | 5% | 10% | 30% | 35% | 55% | 60% | 65% |
  |---|---|---|---|---|---|---|---|---|
  | 特化1 | 8:00:00 | 7:37:09 | 7:16:22 | 6:09:14 | 5:55:33 | 5:09:41 | 5:00:00 | 4:50:55 |
  | 特化2 | 16:00:00 | 15:14:17 | 14:32:44 | 12:18:28 | 11:51:07 | 10:19:21 | 10:00:00 | 9:41:49 |
  | 特化3 | 24:00:00 | 22:51:26 | 21:49:05 | 18:27:42 | 17:46:40 | 15:29:02 | 15:00:00 | 14:32:44 |
  | 1〜3通算 | 48:00:00 | 45:42:51 | 43:38:11 | 36:55:23 | 35:33:20 | 30:58:04 | 30:00:00 | 29:05:27 |
  | 特化3(-50%) | 12:00:00 | 11:25:43 | 10:54:33 | 9:13:51 | 8:53:20 | 7:44:31 | 7:30:00 | 7:16:22 |

**素材消費・不足通知**
- 必要素材は確定時に所持数（BaseOwned）から即時消費する
- 不足時はどの素材が足りないかを、本機能の入力ウィンドウ内にテキストで表示する（ポップアップではない）

**機能要件**
- FR-GR-001: 対象オペレーターを選択し、各強化項目の到達後の値（右側）を入力できること
- FR-GR-002: 現在値（左側）は表示のみとし、入力対象としないこと
- FR-GR-003: 強化実行の確定時に、入力された強化差分（現在値→到達値）に必要な素材を所持数（BaseOwned）から即時消費すること
- FR-GR-004: 強化実行の確定により、対象オペレーターの該当数値（現在状態）を到達値に即時反映すること（スキル特化も確定時に即時反映する）。目標状態とは独立とし、現在値変更に伴い FR-OP-037 の「現在値 ≤ 目標値」ガードを実行すること
- FR-GR-005: 必要素材が不足している場合、どの素材が不足しているかを本機能の入力ウィンドウ内にテキスト表示すること（ポップアップではない）
- FR-GR-006: スキル特化では、スキル番号（1/2/3）・特化レベル（0→1 / 1→2 / 2→3）を選択できること
- FR-GR-007: スキル特化では、訓練時間（短縮率%）と減少時間（-%）を設定できること（数字のフリー入力、可能ならスピンボタン(updown)）
- FR-GR-008: スキル特化の訓練時間は計算式で算出すること（テーブル定義しない。基準: 特化1=8:00:00 / 特化2=16:00:00 / 特化3=24:00:00）
- FR-GR-009: スキル特化の進捗を通知ウィンドウ2（3.6.5）にプログレスバー風で表示すること
- FR-GR-013: 進行中のスキル特化（進捗表示）を、訓練時間の経過を待たずユーザーが任意のタイミングで意図的に終了させられること。終了時は進捗表示をクリアして「訓練室は空です」に戻すこと

#### 3.9.2 累計使用素材の一括表示（おまけ機能）
**概要**: 現在のオペレーター状況に到達するまでに使用した全素材を、初期状態（昇進0・レベル1・スキルレベル1 等）から現在値まで計算して一括表示する。メイン画面とは別ウィンドウで提供する。

**機能要件**
- FR-GR-010: 全オペレーターの初期状態→現在値に要した累計使用素材を計算し、一括表示できること
- FR-GR-011: 累計使用素材の表示はメイン画面とは別ウィンドウで提供すること
- FR-GR-012: 本機能は常時計算せず、ウィンドウを開いたタイミングで計算・表示すること（数秒程度の遅延を許容する）

## 4. 非機能要件

### 4.1 パフォーマンス要件
- NFR-PF-001: 全オペレーター（300体以上）の一覧表示が1秒以内に完了すること
- NFR-PF-002: 素材計算が3秒以内に完了すること
- NFR-PF-003: UI操作に対する応答が0.5秒以内であること（体感的なサクサク感）
- NFR-PF-004: メモリ使用量が500MB以下であること（通常動作時）

### 4.2 ユーザビリティ要件
- NFR-UX-001: 初めて使用するユーザーでも、主要機能を直感的に理解できること
- NFR-UX-002: 頻繁に使用する操作は、3クリック以内でアクセスできること
- NFR-UX-003: エラーメッセージは具体的な対処方法を含むこと

### 4.3 保守性要件
- NFR-MT-001: コードは機能別にモジュール化されていること
- NFR-MT-002: ユーザーデータとマスターデータは明確に分離されていること
- NFR-MT-003: 将来的な機能追加に対応できる拡張性を持つこと
- NFR-MT-004: 調整可能なパラメータ（更新チェックのクールダウン時間、バックアップ間隔・保持世代、ログ保持期間、既定言語・既定テーマ等）は定数として定義し、容易に変更できる実装とすること

### 4.4 信頼性要件
- NFR-RL-001: データ保存失敗時でも、既存データが破損しないこと
- NFR-RL-002: 異常終了時でも、自動バックアップから復旧できること
- NFR-RL-003: マスターデータ更新失敗時でも、アプリが動作を継続できること

### 4.5 互換性要件
- NFR-CP-001: Windows 11以降で動作すること
- NFR-CP-002: .NET 8.0以降のランタイムで動作すること
- NFR-CP-003: 旧バージョンツールのユーザーデータをインポートできること（対応フォーマット・インポート方法は未定。現行ツールの出力フォーマット確定後に定義する）

- CONST-012: インターネット接続は更新チェック時のみ必要とすること

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| オペレーター | アークナイツのキャラクター |
| 昇進 | オペレーターのランクアップシステム（昇進0/1/2） |
| スキル特化 | スキルレベル7以降の強化（マスタリー1/2/3） |
| モジュール | 昇進2以降で強化可能な装備システム |
| 潜在 | オペレーターの強化システム（0～6、0=未所持、1以上=所持） |
| Paradox | 各オペレーター固有のストーリーコンテンツ（逆理演算） |
| マスターデータ | ゲーム公式の固定データ（オペレーター基本情報、素材情報等） |
| ユーザーデータ | ユーザーが入力したデータ（育成状態、所持素材等） |
| 現在状態 | オペレーターの現在の育成状況 |
| 目標状態 | ユーザーが目指す育成目標 |
| 手動入力の所持素材数 | ユーザーが直接入力した素材の所持数 |
| 素材計画 | ゲーム内イベント等で将来入手予定の素材をシミュレーションするための計画 |
| 計画入手素材数 | 素材計画機能で記録した入手予定素材数 |
| 利用可能素材数 | 手動入力分と計画入手分（ON状態のみ）の合計。必要素材計算のインプットとなる |
| 加工割合 | 素材を育成に使用する割合（0～100%）。合成等で消費される分を調整するための機能 |
| 別形態 | 同一オペレーターの異なる形態（例: 近衛アーミヤ）。コードに「_」を含み、親オペレーターの昇進・レベルに依存する |
| 異格 | 同一キャラクターの別オペレーター（例: チェンと遊龍チェン）。独立したオペレーターだが信頼度を共有する |
| 親コード | 別形態オペレーターの元となるオペレーターのコード。「_」より前の部分（例: R001_2 → R001） |
| 無料育成オペレーター | 育成に素材を一切必要としないオペレーター。統合戦略等のゲーム内コンテンツで成長する |
