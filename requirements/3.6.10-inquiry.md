# 3.6.10 アプリ内問い合わせ

## 概要
ユーザーは、ログインやメールアドレス入力なしで、アプリ上から匿名問い合わせを送信できる。問い合わせは「要望」と「バグ報告」に分類し、要望はテキスト中心、バグ報告は必要に応じて診断ログを添付できる。

問い合わせの完全版（本文、診断情報、ログ等）は Cloudflare R2 の非公開 bucket に zip として保存する。public GitHub Issue には公開してよい範囲の要約のみ投稿し、Discord には問い合わせ投稿通知などの簡易通知のみ送信する。

## 入力項目
- 問い合わせ種別
  - 要望
  - バグ報告
- 本文（必須）
- 名前・ニックネーム（任意）
- バグ報告時の発生時期
  - 今日起きた
  - 昨日起きた
  - 数日前から起きている
  - わからない
- バグ報告時の診断ログ添付可否

## 保存・通知方針
- R2 には、問い合わせ本文・任意名・種別・作成日時・診断情報・添付ログを含む完全版を zip で保存する。
- R2 bucket は非公開とし、public GitHub Issue や Discord からログ本体へ直接アクセスできないこと。
- public GitHub Issue には、公開可能な概要、問い合わせID、種別、ログ有無など、個人情報やログ本体を含まない情報のみ投稿する。
- Discord には、問い合わせ投稿通知、種別、問い合わせID、GitHub Issue 番号などの簡易情報のみ送信する。本文全文やログファイルは投稿しない。
- 問い合わせ送信成功の基準は R2 への完全版保存成功とする。GitHub Issue 投稿または Discord 通知に失敗しても、R2 保存に成功していれば問い合わせ本体は受理済みとして扱えること。

## プライバシー・安全性
- メールアドレス欄は設けない。
- 匿名で送信できること、個別返信できないことを送信前に明示する。
- 個人情報や連絡先を本文に入力しないよう注意文を表示する。
- ログ添付はバグ報告時のみ扱う。
- ログには個人情報に該当し得る情報が含まれる可能性があるため、public GitHub Issue と Discord にはログ本体を載せない。
- Cloudflare API key、GitHub token、Discord webhook URL、R2 認証情報などの秘密情報はアプリに含めず、Cloudflare 側の環境変数または binding で管理する。

## 機能要件
- FR-SY-080: ユーザーはアプリ内から問い合わせ画面を開けること。
- FR-SY-081: ユーザーはログインなしで問い合わせを送信できること。
- FR-SY-082: ユーザーはメールアドレスを入力せずに問い合わせを送信できること。
- FR-SY-083: 問い合わせ種別として「要望」と「バグ報告」を選択できること。
- FR-SY-084: 問い合わせ本文は必須入力とすること。
- FR-SY-085: 名前・ニックネームは任意入力とすること。
- FR-SY-086: 匿名問い合わせのため個別返信できない旨を送信前に表示すること。
- FR-SY-087: 個人情報や連絡先を入力しないよう注意文を表示すること。
- FR-SY-088: バグ報告では、発生時期を「今日」「昨日」「数日前」「わからない」から選択できること。
- FR-SY-089: バグ報告では、ユーザーの明示操作により診断ログを添付できること。
- FR-SY-090: 添付ログはアプリ側で対象期間に応じて抽出し、zip 化して送信できること。
- FR-SY-091: 問い合わせの完全版は、Cloudflare R2 の非公開 bucket に zip として保存されること。
- FR-SY-092: R2 に保存する zip には、問い合わせ本文、種別、任意名、作成日時、診断情報、添付ログを含められること。
- FR-SY-093: public GitHub Issue には、公開可能な要約、問い合わせID、種別、ログ有無のみ投稿し、ログ本体や個人情報に該当し得る情報を投稿しないこと。
- FR-SY-094: Discord には、問い合わせ投稿通知などの簡易情報のみ送信し、本文全文やログファイルを投稿しないこと。
- FR-SY-095: GitHub Issue 投稿または Discord 通知に失敗しても、R2 保存に成功した問い合わせ本体は失われないこと。
- FR-SY-096: 本文未入力、文字数超過、ログサイズ超過、送信失敗時には適切なエラーを表示すること。
- FR-SY-097: アプリ側には送信先 API URL 以外の秘密情報を保持しないこと。

## 実装上の前提
- アプリは Cloudflare Pages Functions / Workers の問い合わせ受信 API に POST する。
- Cloudflare 側で入力検証、R2 保存、public GitHub Issue 投稿、Discord webhook 通知を行う。
- R2 object key は問い合わせIDと日付で一意に管理する。
- public GitHub Issue と Discord 通知で使用する payload は、R2 に保存する完全版 payload とは分離する。
